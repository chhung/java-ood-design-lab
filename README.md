## 中階 Java 工程師，逐步深化物件導向與設計模式理解的學習路線圖

✅ 階段一：強化物件導向與 JDK 核心 API 認知  
🎯 目標：  
- 熟練 Java 的物件導向語法與語意（不只是寫得出來，而是寫得對）
- 建立對 JDK 常用類別庫的正確使用認知（如 `java.util`、`java.lang`、`java.io`）

🔧 實作建議：  
- 重構你過去的程式碼，思考封裝、繼承、多型的「正確時機」與「替代方案」
- 嘗試實作一個 帳號系統 / 票券系統，用 `List`, `Map`, `Set`, `Optional`, `Streams` 來處理邏輯
- 深入理解 `Object`、`Class`、`Enum`、`Exception`、`Thread` 的結構與設計哲學

✅ 階段二：導入 SOLID 原則與六大物件導向關係  
🎯 目標：  
- 不只是記憶規則，而是真正內化如何讓「類別結構保持彈性」

🔧 實作建議：  
- 每週選一則 SOLID 原則 實作練習，並寫成程式碼範例 + 自我批註
- 模仿開源專案（例如 Spring、Guava、Apache Commons）如何應用這些原則

✅ 階段三：設計模式 × JDK × 實際應用關聯對照學習  
🎯 目標：  
- 瞭解常見設計模式與 Java 標準類別之間的聯繫

- 🔧 推薦這 6 個模式作為起點，每個都附有 JDK 實例：

| 設計模式      | 概念       | Java JDK 範例                            |
| --------- | -------- | -------------------------------------- |
| Singleton | 控制全域唯一實例 | `Runtime.getRuntime()`                 |
| Factory   | 建立物件的封裝  | `NumberFormat.getInstance()`           |
| Strategy  | 行為切換     | `Comparator` 與 `Collections.sort()`    |
| Observer  | 狀態推播     | `java.util.Observer`（舊）、Spring Events  |
| Decorator | 動態功能增強   | `BufferedInputStream` 包裝 `InputStream` |
| Iterator  | 遍歷集合     | 所有 `Collection` 的 iterator 機制          |


✅ 階段四：用小型專案內化模式與設計原則  
🎯 目標：  
- 把概念真正轉換為實作與設計直覺

- 🔧 專案範例建議（可週週精煉）：
1. 記帳系統：以 MVC 架構寫 RESTful API，導入策略模式控制計算規則
2. 兌換券服務平台：應用觀察者模式通知、工廠模式產生不同券種
3. 快取模組：設計介面 + 裝飾者 + 單例模式，支持快取策略切換

✅ 加分項：你會喜歡的延伸挑戰  
- 從 Java 開始閱讀設計模式原始碼（如 Spring 的 @Transactional 是如何應用代理模式的）
- 自寫一個輕量級的 IoC container（練習 Singleton + Factory + Reflection）
- 嘗試用「註解 + 反射」設計一個 mini framework（加深對 Java 的 runtime 動態能力的認知）
